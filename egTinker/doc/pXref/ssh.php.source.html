  25      /**
  26       *  Execute a single command remotely using ssh and 
  27       * display its output, optionally returning its exit 
  28       * status (like passthru)
  29       *
  30       *    This function is intended to be used as a one-time
  31       * all-at-once non-interactive execution mechanism which
  32       * will run the command remotely and display its output.
  33       *
  34       *    If you try to issue an interactive command using this
  35       * function, all you will get is unneccessary trouble. So
  36       * don't!
  37       *
  38       *  This might be done as well using a pipe on /tmp and
  39       * making the command 'cat' the pipe: when ssh runs, it
  40       * runs the command 'cat' on the pipe and hangs on read.
  41       *  Then we just need a thread to open the pipe, put the
  42       * password and close the pipe.
  43       *  This other way the password is never wirtten down.
  44       * But, OTOH, the file life is so ephemeral that most
  45       * of the time it will only exist in the internal system
  46       * cache, so this approach is not that bad either.
  47       *
  48       *  @param remote   The remote end to run the command, in
  49       *                  the form 'user@host:port' (you may
  50       *                        omit the 'user@' or ':port' parts
  51       *                        if the default values [i.e. same user
  52       *                        or standard port] are OK).
  53       *  @param password The remote password. Note that if direct
  54       *                  RSA/DSA/.shosts/.rhosts login is enabled
  55       *                  then the password should be ignored as
  56       *                  SSH should not run the ASKPASS command).
  57       *  @param command  The command to execute on the remote end
  58       *                  NOTE: if you want to use redirection, the
  59       *                  entire remote command line should be 
  60       *                  enclosed in additional quotes!
  61       *  @param status   Optional, this will hold the termination
  62       *                  status of SSH after invocation, which
  63       *                  should be the exit status of the remote
  64       *                  command or 255 if an error occurred
  65       *  @return void
  66       */
